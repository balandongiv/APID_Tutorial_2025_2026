Relevant Editorial Feedback (Mapped)

Missing or malformed code examples. A dictionary introduction example in this session shows sample = followed by key/value lines but omits curly braces. This will confuse beginners because it looks runnable but is not valid Python.

Context drift and mixed metaphors. The chapter mentions robots when explaining dictionaries, even though the course is about flowers. This unnecessary domain switch distracts beginners and should be removed.

Session/Chapter mismatch and file references. It’s unclear which files belong to this session. A consistent “Hands‑on files” box needs to list only session3/session3_iris_template.py and show how to run it.

Self‑learning lists. The text tells students to “read about list on your own” right before lists become critical. Beginners need at least a minimal explanation of lists, indexing, and mutability to succeed.

Embedded solutions. A lecturer-only solution block for the second flower is present. Leaving full answers in the student version undermines practice.

Metrics logic bug. The editorial feedback warns that in later sessions, ClassifierBase.evaluate increments total only when predictions are correct. Session 3 introduces accuracy counters, so ensure total increases for every sample (even wrong predictions) to avoid this pitfall.
